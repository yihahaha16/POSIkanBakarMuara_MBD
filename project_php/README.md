Studi kasus ini adalah Sistem Pemesanan Restoran Ikan Bakar Muara Sistem ini dibuat agar pelanggan dapat melakukan pemesanan secara mandiri.
Pada halaman index, pelanggan akan diminta untuk mengisi informasi data pelanggan, seperti nomor telepon dan nama. Lalu dilanjutkan untuk mengisi data pesanan, seperti tanggal pesan, nomor meja, jenis pesanan, dan menyesuaikan kuantitas/jumlah menu yang diinginkan. Data ini akan disimpan pada database menggunakan PDO dan PHP. 
Selanjutnya, pelanggan akan dialihkan ke halaman tambah. Halaman ini akan menampilkan menu yang telah dipilih oleh pelanggan serta detail pesanan, seperti kuantitas yang dipesan, subtotal per menu, dan grand total pesanan. Pelanggan dapat melakukan aksi, seperti mengubah kuantitas dengan tombol edit dan menghapus pesanan dengan tombol hapus. Pelanggan akan diarahkan ke halaman edit atau hapus jika menekan tombol-tombol tersebut. Pada aksi edit, pelanggan dapat mengubah kuantitas menu yang telah dipesan dengan syarat kuantitas tidak boleh <= 0. Sedangkan pada aksi hapus, pelanggan dapat menghapus menu yang dipesan dengan syarat melakukan konfirmasi terlebih dahulu (ya/tidak). Dengan adanya sistem ini, proses pencatatan akan lebih efisien dan terstruktur, sehingga pelanggan dapat melakukan pemesanan dengan lebih mudah.
